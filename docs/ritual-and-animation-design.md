# 仪式感与动画设计建议

更新时间：2026-06-29

## 背景与定位

本文与 [`romantic-two-person-feature-strategy.md`](./romantic-two-person-feature-strategy.md) 互补。那份文档回答"做什么功能"，本文回答"这些时刻如何被感受到"——聚焦**仪式感、动画效果与设计细节**，从情侣用户的真实情绪出发。

Our Memories 已经有一套相当精致的设计基底：

- 登录页 `EntryExperience.tsx` 有视差指针跟随（`driftX`/`driftY`）、手绘地图路线绘制（`entryRouteDraw`）、拍立得弹窗（spring 过渡）、解锁后"正在打开地图"遮罩。
- 城市标记 `CityMarker.tsx` 点亮时用 spring 从灰转 `bloom` 并带辉光；地标 `LandmarkSprite.tsx` 灰度→彩色过渡。
- `globals.css` 有一套完整的像素风暖色体系（sakura/bloom/mist/cream）和关键帧（`entryPulse`/`entryFloat`/`entrySweep`）。
- `HomeProgress.tsx` 有"在一起 N 天"、省份/城市进度条（带辉光）、像素花、可点击放大的双人拼图头像、像素天气图标。

但也存在几处"情绪断点"——内容到了最该有仪式的时刻反而变得扁平。本文按用户旅程逐一指出，并给出可落地的动画/交互建议。

## 设计原则

1. **仪式 ≠ 装饰堆砌**。动画要服务于"这一刻很重要"，而不是每个元素都动。克制是浪漫的一部分。
2. **让等待被感受到**。情侣产品里，延迟（胶囊、纪念日）本身就是情绪价值。倒计时应该有"越来越近"的体感，而不是一个静态数字。
3. **第一次最珍贵**。第一次点亮一座城、第一次开启胶囊、第 100/365 天——这些里程碑值得被单独庆祝。
4. **触感与声音是可选的增强层**，默认克制，尊重"私密"语境；但移动端轻微震动在关键时刻能显著提升"真实感"。
5. **氛围是底噪，不是主角**。漂浮的花瓣/雪花/光点应当极轻，绝不抢走照片和文字。
6. **复用现有视觉语言**。项目已有像素风 + 暖色 + spring 动效，新增动效应延续这套语汇，避免引入割裂的风格。

## 一、开门：登录仪式

现状：`EntryExperience.tsx` 已经很好。密码输错有抖动（`x: [-8,8,-6,6,0]`），解锁后有 heart 遮罩。

可增强：

- **密码点亮的"心跳"节奏**。`login-code-dot.is-filled`（`globals.css:273`）目前是即时变色。可以让第 4 位填满时四个点同步一次轻脉动（`box-shadow` 扩散），像"门锁咬合"。
- **解锁瞬间的"开门"过渡**。当前 `status === "open"` 直接覆盖一个 heart 遮罩（`EntryExperience.tsx:527`）。可在遮罩出现前加一帧"门缝透光"：一条从中间向两侧展开的高光带，再淡入 heart，强化"推开回忆的门"。
- **选择身份（me/ta）时的双人呼应**。`step === 2` 选择"刘永伦/郭文盈"（`EntryExperience.tsx:359`）时，被选中方可有一次轻微的"靠近"位移，暗示"两个人一起进来"。

## 二、记录：写下回忆的仪式

现状：回忆编辑分散在 `useMemoryEditor.ts`、`AddMemoryPanel.tsx`、`MemoryCitySheet.tsx`，保存后仅一个 toast。

可增强：

- **保存成功的"收藏进回忆"微仪式**。替代/补充当前 toast：保存后让该条回忆卡片向相册方向"飞入"并淡出，配一句"已收进我们的回忆"。低成本、高情绪。
- **partner note 的"TA 补充了这一天"惊喜**。`MemoryDetailSheet.tsx` 的 `partnerNote`（`MemoryDetailSheet.tsx:19`）目前直接作为文本展示。当对方刚补充、且这是你首次看到时，应先展示一张"TA 在这一天留下了话"的封口卡片，轻触后才展开内容——把"读取"变成"拆信"。
- **照片上传的"显影"效果**。`AnniversaryWall.tsx`/`time-capsule/page.tsx` 上传后照片直接出现。可加一帧从模糊+低饱和到清晰的"显影"过渡，呼应登录页"城市像一张慢慢显影的照片"的文案。

## 三、点亮：城市的第一次

现状：`CityMarker.tsx` 点亮用 spring + 辉光，`LandmarkSprite.tsx` 灰度→彩色。已经不错，但"第一次点亮"和"第 N 次点亮"没有区别。

可增强：

- **首次点亮的庆祝时刻**。检测某城市从无回忆→有回忆的转折，触发一次"地标像素逐块点亮"动画（`LandmarkSprite` 可分块淡入），并弹出克制的 toast："第 N 座城市点亮 · 城市名"。这是情侣地图最该有的高光。
- **点亮时的粒子**。首次点亮可撒一把 sakura 花瓣（复用 `HomeProgress` 的 `PixelFlower` 配色），spring 散开即消，不循环。
- **省份集齐的里程碑**。`HomeProgress.tsx` 的 `progress.provinceCount`（`HomeProgress.tsx:467`）到达整数门槛（如 10/20/34）时，进度条可有一次"流光满格"动画 + 一句鼓励语。

## 四、等待：时光胶囊的倒计时

现状：这是**最大的仪式感缺口**。`time-capsule/page.tsx:292` 锁定时仅展示一个静态数字"X 后天开启"；移动端 `MapTimeCapsules.tsx` 有渐变锁状胶囊条但无倒计时动效。`romantic-two-person-feature-strategy.md` 已把"开箱仪式"列为 P0。

可增强（按接近程度分层强化）：

- **倒计时体感随距离递增**：
  - 远期（>7 天）：保持现状，安静的锁。
  - 临近（≤7 天）：锁的边框开始轻微呼吸（复用 `entryPulse`），数字旁加"快到了"。
  - 当天（=0 天，未开）：明显的心跳节律（每 1.2s 一次脉动），入口高亮，"今天就到了"。
- **地图入口的"今日到期"提示**。`MapTimeCapsules.tsx` 只展示未到期胶囊。到期当天应把该胶囊置顶并加"今日开启"标记，引导用户去开启。
- **开启仪式（核心）**。当前开启后直接渲染 `cap.content` + "✓ 已打开"（`time-capsule/page.tsx:308`）。建议改为多阶段：
  1. 用户点击"开启"。
  2. 锁芯碎裂/消散动画（像素块飞散，复用像素风）。
  3. 一束光从中心透出，内容随光从上到下"显影"。
  4. 落款"埋于 YYYY-MM-DD · 由 XX 埋下"作为收尾印章。
- **"一起打开"模式**（对应策略文档 `open_mode: together`）：到期后需要双方都点确认。一方点完后，另一方看到的是"TA 已经在等你一起打开"，两人都确认后才触发上面的开启动画。

## 五、纪念：纪念日的"今天"

现状：`AnniversaryWall.tsx` 的 `anniversaryDisplayState` 计算"距今/下一次"（`AnniversaryWall.tsx:343`），但当今天就是纪念日时，没有特别的"今天就是这天"的高光。

可增强：

- **当天的英雄卡片**。若 `state.daysUntil === 0`（或 repeat 当日），把该卡片提升为顶部 hero：大字号"今天是我们的第 N 个 XX"，配一张代表照片，加一次轻柔的彩屑/花瓣。
- **"那天我们经历了什么"回放**（对应策略文档 P1 纪念日自动回放）。当天聚合同日期/相近日期的旧回忆、当时去过的城市、写下的文字，做成只读回放视图，并提供"补一句今年的感受"入口。
- **倒数临近的"快到了"**。`state.daysUntil` 在 1–3 天时，卡片角落可加一个轻微呼吸的提示，避免重要的日子在不知不觉中滑过。

## 六、在一起：天数的里程碑

现状：`HomeProgress.tsx` 的 `TogetherDaysCard`（`HomeProgress.tsx:426`）和 `MobileRitualStats`（`HomeProgress.tsx:619`）展示天数，但数字是静态渲染。

可增强：

- **数字滚动入场**。`days` 在首屏用一次 count-up（0→N，约 800ms，缓出），让"在一起"被看见而不是被读出。
- **里程碑庆祝**。100/200/365/500/1000 天等触发一次轻庆祝：数字弹一下 + 一句"第 N 天，继续走下去" + 可选花瓣。仅在该里程碑首次出现当天触发，不重复打扰。
- **周年与天数联动**。若当天恰好是周年，天数卡片与纪念日英雄卡片应互相呼应（同一配色高光），而不是两张各管各的卡。

## 七、悄悄话：私密对话的温度

现状：`WhisperWall.tsx` 是项目里**最朴素**的页面——灰色卡片、无动画、无温度，与登录页的精致反差明显。悄悄话本应是"两个人之间的秘密纸条"。

可增强：

- **纸条入场**。新 whisper 用一次"从下方滑入 + 轻微旋转"进入（复用登录页 `login-polaroid` 的 spring 风格），而不是直接出现在 grid 里。
- **未读的"封口"感**。对方新发的、你还没看的悄悄话，先以封口信封形态展示，轻触"拆开"再展开内容——把阅读变成动作。
- **"正在想你"的轻触信号**（对应策略文档"想你信号"）。在地图某城市或首页一个心形按钮轻触一次，对方收到"TA 刚刚想到了这里"的瞬时信号，几秒后自动消散，不留未读压力。
- **书写时的墨水感**。`Textarea` 输入时可加一帧极轻的"墨迹晕开"边框反馈，让打字也有手感。

## 八、回看：旧内容的浮现

现状：旧回忆通过 `MemoryDetailSheet.tsx` 的标准 `BottomSheet` 打开。

可增强：

- **"拆旧信"的展开**。打开一条 N 天前的回忆时，顶部加一个日期邮戳（复用登录页 `login-stamp` 风格），内容区用纸质底（`--color-parchment` 已存在），slow reveal。
- **"X 天前的今天"主动浮现**。首页可加一条极轻的"一年前的今天"入口（若有），点开即回放当时回忆。低成本、高情感。
- **照片灯箱的仪式**。`PhotoLightbox.tsx` 打开时可加一帧"老照片显影"（从低饱和模糊到清晰），呼应"旧照片新地图"的主题。

## 九、氛围层（Ambient）

现状：页面在交互之间是静止的。`HomeProgress` 已经拉取了天气数据（`WeatherCard`），但未用于氛围。

可增强：

- **天气驱动的环境粒子**。根据当前天气 `WeatherKind`（`HomeProgress.tsx:64`）在地图/首页撒极轻的环境元素：晴=光斑、雨=细雨线、雪=雪花、夜=星点。透明度极低、数量极少，纯粹是底噪。
- **时段驱动的色温**。登录页已有 `login-sun`。可把这套思路延伸到全局：日落时分整体偏暖、夜晚偏冷蓝（用一层极轻的 `mix-blend` 覆盖），让"什么时候打开"也有体感。
- **季节性的像素装饰**。春 sakura、秋落叶——在 `globals.css` 已有的像素风基础上加季节性小元素，量要少。

## 十、过渡与微交互

现状：`PageTransition.tsx` 是统一的 fade + x:20 slide，所有页面相同。

可增强：

- **目的地化的页面过渡**。按路由分主题：进地图=路线绘制感、进相册=照片翻转、进胶囊=锁的溶解、进悄悄话=纸条滑入。只需在 `PageTransition` 里按 `pathname` 切换 `variants`。
- **下拉刷新的主题指示**。`PullToRefresh.tsx` 可用一个"心形逐渐填满"或"邮戳逐渐盖印"的指示器替代默认 spinner，呼应产品主题。
- **FAB 的呼吸**。`AnniversaryWall`/`WhisperWall`/`time-capsule` 的悬浮 `+` 按钮在有"可写"内容时可加极轻呼吸，引导记录。
- **触感反馈（移动端）**。点亮城市、开启胶囊、保存回忆、双人头像 pop（`HomeProgress.tsx:563`）这些时刻加 `navigator.vibrate(10)` 级别的轻震，默认开启但可在设置里关。

## 实施顺序建议

1. **先补"开启仪式"和"悄悄话温度"**——这两处是当前情绪断点最明显的地方，且与策略文档 P0 完全对齐，投入产出比最高。
2. **再做"在一起天数"与"首次点亮"**——它们是高频可见的里程碑，能让日常使用就有仪式感。
3. **然后做"纪念日当天"与"partner note 拆信"**——把已有的双人/时间数据变成"被记得"的体验。
4. **最后做氛围层、过渡主题、触感**——这些是锦上添花，应在核心仪式稳了之后再上，避免分散注意力。

## 注意事项

- **性能**。粒子/呼吸动画务必 `will-change` 限定范围、数量极少、优先 CSS 动画与 transform/opacity，避免在地图（`ProvinceMapCanvas.tsx` 已较重）上叠加重负担。
- **可访问性**。所有装饰性动画应尊重 `prefers-reduced-motion`，为减少动效用户降级为即时显示。
- **可关闭**。氛围粒子、触感、声音这类增强层应在设置里提供开关，默认克制，符合"私密"产品气质。
- **不要过度游戏化**。仪式感来自"这一刻被认真对待"，而非连续签到、排行榜——这与策略文档"不建议优先做的方向"一致。
