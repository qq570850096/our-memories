# 情侣视角的动效、仪式感与新功能点

更新时间：2026-06-29
视角：站在两个人使用 Our Memories 的角度，而不是产品经理排期的角度
参照代码：`apps/web/components/EntryExperience.tsx`、`PageTransition.tsx`、`RecentMemories.tsx`、`ProvinceMap/`、`memories/MemoryDetailSheet.tsx`、`WhisperWall.tsx`、`AnniversaryWall.tsx`

## 为什么不从产品经理视角写

产品经理视角容易把"浪漫"拆成功能清单：双人视角、时间胶囊、路线地图。这些当然对，但写出来像排期表，不像两个人在用的东西。

真正打动情侣的，往往是那些"没什么用但让人心里一软"的瞬间：对方刚刚来过、旧照片突然亮起来、那天风很大、密码是我们认识的那天。这些瞬间不是功能，是**被设计出来的感受**。

所以这份文档按"两个人在一起会经历的瞬间"来组织，每个瞬间下面写：该有什么动效、该有什么仪式感、可以落到哪个组件上。

## 端优先级：手机先行

这份文档初版是从 `apps/web`（桌面/大屏）视角写的，很多动效默认了 hover、鼠标、宽屏 SVG。但情侣记录回忆的真实场景绝大多数在手机上：地铁里、睡前、出差路上、对方发来一条微信后顺手打开。所以下面所有瞬间的落地，都要先过"手机端能不能成立"这一关，桌面端是放大版而非基准版。

### 手机优先的几条硬约束

1. **没有 hover**。触屏只有 tap 和 long-press。凡是在 web 上靠 `onHoverStart` 预览的（`ProvinceMapCanvas` 现在就是这样），手机端必须改成"点击即选中、再点展开详情"，或者用 long-press 唤起次级动作（如想你信号）。文档里凡是写"鼠标 hover 时显示"的地方，都理解为"点击/长按时显示"。

2. **单手拇指可达**。`/map` 顶部和底部边缘是拇指热区，中间是冷区。仪式性入口（胶囊浮起、纪念日回放卡、想你信号触发）优先贴底或贴边；详情 sheet 从底部上滑，不要从侧边或中央弹出。

3. **慢动效要更克制**。桌面端可以接受 2 秒的路线绘制、1.5 秒显影；手机端用户常常在碎片时间用，超过 0.8 秒的全程阻塞动画会让人想退出。原则：慢动画只用在"主动打开"的仪式场景（胶囊显影、纪念日回放），日常浏览（列表、地图切换）保持 0.2-0.4 秒。

4. **弱网和断连是常态**。地铁、电梯、异地漫游。所有"对方刚来过"的实时感不能依赖长连接不断——必须能离线降级：WS 断了就退化为"上次同步的痕迹"，不要转圈等待。悄悄话"正在写"在弱网下宁可静默，也不要显示假状态。

5. **推送靠系统/小程序，不要自造**。纪念日、胶囊到期这类"不在应用内也要触达"的提醒，手机端走微信小程序订阅消息或 Capacitor 本地推送，不要妄想靠应用内通知。情侣不会为了一个回忆应用常驻后台。

6. **分享走系统级**。周年长图、纪念日回放，手机端的终点是"保存到相册 / 转发到微信"，不是 html2canvas 截屏发邮件。导出功能要先确认能落进系统相册和微信分享面板。

### 落到组件上

- `ProvinceMap` 系列：保留 hover 给桌面，但所有 hover 暴露的信息在手机端必须有 tap 等价路径。`useProvinceMapGeometry` hook 要同时返回"命中区域几何"和"长按命中区域几何"两套。
- `EntryExperience`：密码输入在手机端用系统数字键盘，圆点心跳动效要适配小屏；首图按时段切换在手机端更要省流量（清晨图可能要 webp 压缩版）。
- `MemoryDetailSheet` / `MemoryCitySheet`：已经是底部 sheet，手机原生体验对，桌面端再做成居中 modal 即可。
- 小程序端（`apps/miniprogram`）：很多"仪式瞬间"在小程序里只能做简化版（无法自定义动效到 framer-motion 级别），优先保证"信息到达"——纪念日卡、胶囊到期提示、想你信号接收，动效退化为淡入淡出。

## 瞬间一：开门那一刻

### 现状

`EntryExperience.tsx` 已经做得很克制：飘云、视差漂移、宝丽来旋转入场、密码输错时整块抖动。但开门的"仪式感"还可以更深。

### 动效建议

**1. 密码输入时的呼吸**
现在是 4 个圆点填满就提交。可以让圆点在填满时有一个轻微的"心跳"放大再回弹，像确认"你记起来了"。输错时整块横向抖动已经有了，可以再加一个圆点短暂变暗再恢复，像叹了一口气。

**2. 第二步选身份时的双人呼吸**
现在是两个按钮 `刘永伦` / `郭文盈`。选中后对方那个按钮可以有一个非常轻微的"熄灭"动效（透明度降到 0.6、轻微缩小），像"另一个我暂时不在线"。这比单纯高亮选中态更有"我在这一边"的感觉。

**3. 解锁瞬间的"开门"**
现在 `status === "open"` 时弹出一个心形 + "正在打开地图"。可以让整个画面先有一个 0.3 秒的过曝白光（像推开门时阳光涌进来），再淡入心形，再路由跳转。白光用一层 `bg-white` 配 `opacity` 动画即可，成本很低但仪式感强很多。

**4. 首图随时间变化**
`loginStamps` 现在是固定的 9 张。可以根据当前小时段选首图：清晨用"济南泉边"、傍晚用"上海外滩"、深夜用"香港夜色"。不需要新建数据，从现有 stamps 里按 `label` 或新增一个 `time` 字段映射即可。让"我们打开应用的时间"和"看到的画面"产生关联。

### 仪式感

- 密码是纪念日，本身就是仪式。可以在输入框上方那行小字 `一扇只给我们的回忆门，密码藏在开始的那一天。` 下面，加一句会变的话：如果今天是某个纪念日前后 3 天，显示"距离我们第 N 个纪念日还有 X 天"；如果已经过了，显示"这是我们在一起的第 N 天"。用 `AnniversaryCard` 数据 + 当前日期计算。

### 落点

- `EntryExperience.tsx` 的 `pressKey`、`submitCode`、`status` 状态机
- 新增一个 `useAnniversaryProximity` hook，从 `/api/v1/anniversary-cards` 拉数据算天数

## 瞬间二：知道对方刚刚来过

### 现状

目前没有"对方在线/最近活跃"的任何痕迹。两个人用同一个空间，但彼此不知道对方什么时候来过、看了什么、留下了什么。这是当前最大的情感空缺。

### 新功能：足迹痕迹（不是实时位置）

不是实时定位（那会破坏信任感），而是"对方最近 24 小时在这个空间留下了什么痕迹"。

**形式一：首页角落的微光**
进入 `/map` 后，如果对方在过去几小时内编辑过某条回忆、回复过悄悄话、创建过胶囊，那座城市在地图上会有一个非常轻微的呼吸光晕（不是红点，是暖色微光）。桌面端鼠标 hover 显示"TA 刚刚补充了杭州的回忆"；手机端没有 hover，改为点击该城市时在 sheet 顶部先出现这行字，再展示回忆列表——让"TA 来过"成为打开城市的第一句话，而不是需要悬停才发现的彩蛋。

**形式二：详情页的"TA 也看过"**
打开一条回忆详情 `MemoryDetailSheet`，如果对方在过去 24 小时打开过这条，底部出现一行极小的字："TA 昨晚看过这条"。不需要已读回执的压迫感，只是一句安静的话。

**形式三：悄悄话的"正在写"**
`WhisperWall` 现在是发出去等回复。可以在对方正在输入时，那条悄悄话卡片底部出现三个缓慢跳动的点。成本是一点 SSE 或轮询，但情感价值很高——异地情侣最想要的就是"TA 在"。

### 动效

- 地图城市微光：在 `ProvinceMap` 的城市节点上叠一层 `motion.div`，`animate={{ opacity: [0.3, 0.7, 0.3] }}` 循环，`transition={{ duration: 3, repeat: Infinity }}`，只在有新痕迹的城市出现。
- "TA 也看过"：底部一行字从下方 `y: 8, opacity: 0` 滑入，停留，不消失。
- 正在写：三个点用 `staggerChildren` 错开 0.2 秒上下跳动。

### 落点

- 后端新增 `activity` 概念：在 `events.Dispatcher` 里已经有 `MemoryCreated`、`WhisperReplied` 等事件，增加一个 `MemoryViewed` 事件，写入一个轻量的 `recent_activity` 表（space_id、user_id、target_type、target_id、seen_at），TTL 24 小时。
- 前端 `ProvinceMap` 读取该 space 的 recent activity，渲染微光。
- 悄悄话"正在写"需要轻量实时通道，见技术文档。

## 瞬间三：等待的重量

### 现状

`TimeCapsule` 已经有 `OpenDate` 和 `IsOpened`。但"等待"目前只是一个倒计时数字，没有重量。

### 仪式感设计

**1. 创建时的"封存"动作**
现在创建胶囊就是填表提交。可以加一个封存动画：提交后，内容区域从下往上被一层"封蜡"颜色覆盖（用 `clip-path` 动画），最后盖一个只有两个人知道的印记（比如你们第一次见面的日期戳）。这个动画只播放一次，然后跳回列表。

**2. 到期当天的"今天可以打开了"**
不要用红点提醒。在 `/map` 首页底部放一个半埋在画面边缘的胶囊图标，平时很淡，到期当天它会慢慢浮起来一点、透出一丝暖光。点进去才看到完整内容。

**3. 打开时的"显影"**
现在 `OpenTimeCapsule` 应该是直接返回内容。前端打开时不要瞬间展示全文，而是像宝丽来显影：照片先出现模糊轮廓，用 1.5 秒慢慢变清晰；文字一个字一个字浮现（用 `staggerChildren` 按字符延迟）。这种"慢慢看清"才是仪式。

**4. 一起打开模式**
新增 `open_mode: "together"`：到期后需要两个人都点过"我也准备好了"，才一起显影。对方还没点时，先点的人看到的是"TA 还在路上"。这比单人打开浪漫得多，特别适合异地。

### 动效

- 封存：`motion.div` 覆盖层 `initial={{ scaleY: 0 }} animate={{ scaleY: 1 }} transformOrigin: bottom`，0.8 秒。
- 浮起：胶囊图标 `animate={{ y: [20, 0], opacity: [0.4, 1] }}` spring。
- 显影：照片 `filter: blur(20px) → blur(0px)` over 1.5s；文字按字符 `staggerChildren: 0.03`。
- 一起打开：两人头像在胶囊两侧，都点亮后中间光线连起来（一条 `pathLength` 从 0 到 1 的 SVG 线）。

### 落点

- `timecapsule_service.go` 增加 `open_mode`、`opened_by_user_ids`、`revealed_at` 字段
- `handlers/timecapsule.go` 的 `OpenTimeCapsule` 改为记录"谁点了"，两人都点后才返回 content
- 前端新建 `TimeCapsuleReveal` 组件承载显影动画

## 瞬间四：重逢某一天

### 现状

`AnniversaryWall` 展示纪念日卡片，但没有"回到那天"的能力。`RecentMemories` 是随机推荐，和日期无关。

### 新功能：纪念日回放页

纪念日当天（或前后 3 天）进入应用，首页顶部出现一张"今天是我们第 N 个某某日"的卡片，点进去是一个只读回放页：

- 顶部：那天的日期、当时的城市
- 主体：那天拍的照片（从 `Memory` 里按 `date` 匹配）、那天的文字
- 底部：一行"今年想补一句吗？"，点击展开一个极简输入框，保存为一条新回忆，关联到这个纪念日

这不是"倒数还有几天"，是"那天我们经历了什么"。

### 动效

- 回放页入场：整页从一张老照片的局部开始，缓慢拉远（`scale: 1.2 → 1` over 2s），像从记忆里抽身出来。
- 照片墙：那天的照片用"翻牌"动画依次出现，每张间隔 0.4 秒。
- "今年想补一句"：输入框默认收起为一个带光标闪烁的横线，点击后展开。

### 落点

- 后端新增 `GET /api/v1/anniversary-cards/:id/replay`，聚合该日期 ±3 天的 memories
- 前端 `app/anniversaries/[id]/replay/page.tsx`
- `AnniversaryWall` 卡片在临近日期时显示"回放"入口

## 瞬间五：不用说话也在

### 现状

`WhisperWall` 是文字悄悄话，需要写内容。但有时候只是想"戳一下"，不想打字。

### 新功能：想你信号（极轻量）

在 `ProvinceMap` 某座城市上长按，弹出一个心形按钮，点一下就给对方发一个"TA 刚刚想到了这里"。对方下次打开地图时，那座城市有一颗心跳一下就消失（存在 24 小时）。

可以附带一句不超过 10 个字的短句，从预设里选："突然想到那天的风"、"想你了"、"这边下雨了"。

### 动效

- 发送：心形从手指位置 `scale: 0 → 1` 弹出，然后飞向城市坐标方向淡出。
- 接收：对方地图上那座城市出现一颗心，`scale: [0, 1.2, 1]` 后停留 2 秒，再 `opacity: 1 → 0` 缓慢消失。不弹窗、不震动、不红点。

### 仪式感

- 不做连续签到、不做"今天想了几次"统计。信号就是信号，出现过就够。
- 信号只对双方可见，永远不进管理后台的统计。

### 落点

- 新增 `relationship_signals` 表：`space_id`、`sender_user_id`、`city_id`、`message`、`created_at`、`expires_at`
- `events.Publisher` 发 `SignalCreated` 事件
- 前端 `ProvinceMap` 叠一个 `SignalLayer`

## 瞬间六：一起走过的路

### 现状

`ProvinceMap` 展示城市点，`MapTimeCapsules` 在地图上叠胶囊。但没有"路线"概念，城市是散落的点。

### 新功能：我们的路线

把城市按 `Memory.Date` 时间排序，在地图上用一条虚线串起来。不是导航路径，是时间路径。线的粗细可以按那个城市的回忆数量变化（去过 5 次的城市线更粗）。

可以切换三种视图：
- **时间线**：按第一次记录的日期排序
- **故事线**：手动标记的节点（初见、表白、第一次旅行、搬家、周年），节点之间用线连
- **主题线**：按 `Memory.Tags` 筛选，比如"看海的城市"、"吃过的城市"

### 动效

- 路线绘制：SVG `path` 用 `pathLength: 0 → 1` 在 2 秒内画完，像在回忆里走了一遍。
- 节点出现：每个城市点在线画到时才 `scale: 0 → 1` 弹出。
- 切换视图：旧线 `opacity: 1 → 0` 淡出，新线重新绘制。

### 仪式感

- 路线终点不是最新的城市，而是"下一个想去的地方"——一个虚线圆圈，写着"待续"。两个人都可以往里加城市，但只能加一个，加了之后对方会看到"TA 想去这里"。
- 这比任何 TODO 清单都浪漫。

### 落点

- 复用现有 `Memory.Date` + `CityID` 派生默认路线
- 新增 `memory_route_nodes` 表存手动标记的节点
- `ProvinceMap` 抽出 `useProvinceMapGeometry` hook 承载路线几何计算，避免组件继续变重（`OPTIMIZATION_GUIDE.md` 已经提过这个方向）

## 瞬间七：只属于我们的暗号

### 现状

`Memory.Tags` 是普通标签，UI 上就是几个灰色 chip。

### 新功能：暗号标签

把标签叫"暗号"。允许每个暗号补一句只有两个人懂的解释，默认对外不可见（其实本来就没有外人）。在地图和回忆列表里可以按暗号筛选。

预设几个浪漫的暗号模板：
- "蓝色便利店"
- "雨停之后"
- "那天的风"
- "不许忘记"

### 动效

- 暗号 chip 用手写体字体（`font-family` 引入一个手写 webfont），轻微旋转 `-2deg`。
- hover 时展开那句解释，从下方 `y: 8, opacity: 0` 滑入。
- 筛选时，不含该暗号的城市先 `opacity: 0.3` 变淡，而不是直接消失。

### 落点

- `Memory.Tags` 字段已存在，只需前端 UI 升级 + 可选的 `tag_notes` JSON 字段存解释
- `MemoryArchive`、`ProvinceMap` 的筛选逻辑

## 瞬间八：每年的礼物

### 现状

照片和回忆都是零散的，没有"年度总结"。

### 新功能：周年影片/长图

到关系周年时，自动生成一份"这一年"：
- 这一年去过的城市（按时间排序的小地图）
- 这一年拍的第一张和最后一张照片
- 这一年写过的最长一段文字
- 这一年创建的胶囊数量
- 两个人各贡献了多少条回忆

### 动效

- 生成时有一个"洗照片"的动画：几张代表照片在流水里慢慢显影，最后拼成一张长图。
- 长图可以保存到相册，或者作为一个特殊的 `Memory` 存档。

### 落点

- 这是高成本功能，适合付费增值（终身版 ¥99 的卖点之一）
- 后端聚合查询；前端在 web 端可用 `html2canvas`，但手机端（Capacitor / 小程序）的终点是"保存到系统相册 + 转发到微信"，必须走端原生分享，不要依赖 web 截图方案
- 不需要 AI 生成文案，用真实数据最有力量

## 贯穿性的动效原则

1. **慢一点**
   现在的动效普遍在 0.2-0.5 秒。回忆类应用应该允许更慢：显影 1.5 秒、路线绘制 2 秒、封存 0.8 秒。慢本身就是仪式。

2. **不要弹窗**
   所有提醒都用画面内的元素承载，不用 `alert`、不用 toast 红点。微光、浮起、淡入，比弹窗温柔。

3. **重复才有仪式**
   每次解锁都过曝白光、每次封存都盖蜡、每次显影都模糊到清晰——重复的动作会变成两个人的共同记忆。

4. **留白比信息重要**
   `RecentMemories` 现在信息密度合适。新功能不要把首页填满，宁可少几个入口，让每个入口都有呼吸空间。

5. **声音可选**
   可以加一组极轻的环境音开关：开锁声、显影声、心跳声。默认关闭，需要的人自己开。声音是仪式感最便宜的放大器，但必须可关。

## 优先级（从情侣感受出发，不是技术成本）

| 优先级 | 瞬间 | 为什么 |
| --- | --- | --- |
| 最先做 | 知道对方刚刚来过 | 这是当前最大的情感空缺，两个人用同一个空间却互不可见 |
| 最先做 | 等待的重量（胶囊仪式） | 胶囊已经存在，只差仪式化，成本最低收益最高 |
| 接着做 | 重逢某一天（纪念日回放） | 复用现有数据，情绪价值极高 |
| 接着做 | 不用说话也在（想你信号） | 异地场景刚需，极轻量 |
| 之后做 | 一起走过的路（路线） | 视觉冲击大但需要路线数据梳理 |
| 最后做 | 每年的礼物（周年影片） | 高成本，适合付费卖点 |

---

## 一句话总结

Our Memories 现在是一个"很好用的回忆相册"。要变成"两个人离不开的关系容器"，缺的不是功能，是那些让两个人在使用时心里一软的瞬间：开门时的光、对方来过的微光、胶囊慢慢显影、纪念日那天回到那天。这些瞬间单个都很小，叠在一起就是"这个应用懂我们"。
